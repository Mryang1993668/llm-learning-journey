# 🎯 微调技术 LoRA & QLoRA

> **基于公司知识助手实战**

## 📝 一句话理解

**LoRA和QLoRA就是给大模型开“外挂补习班”的技术**，让模型真正学会你的业务知识，而不是像RAG那样每次现查资料。

## 🧠 微调 vs RAG：两种让模型“懂你”的方式

结合我刚做完的**公司知识助手**实战，对比一下就清楚了：

| 维度 | RAG（我已掌握） | 微调（LoRA/QLoRA） |
|------|----------------|-------------------|
| **打个比方** | 给模型配个“图书馆”，每次问问题现查书 | 让模型把书“背下来”，考试时直接答 |
| **工作原理** | 不改变模型，只是临时给它资料 | 改变模型内部的权重，让它真正学会 |
| **数据形式** | 任何文档都能直接上传 | 需要整理成“问题-答案”对格式 |
| **回答方式** | 依赖检索，检索不到就瞎编 | 直接记住知识，回答更快 |
| **知识更新** | 换文档就行，不用重新训练 | 知识变了必须重新训练 |
| **硬件需求** | 几乎为零，普通电脑就能跑 | **需要显卡**，配置越高越快 |
| **效果质量** | 依赖检索质量 | 模型真正变成领域专家 |

### 用公司知识助手举例：
````
**RAG方式**：

用这些数据训练2小时（模型真的记住了）
↓

↓
模型直接答：五险一金全额缴纳、餐补30元/天...
（不用去检索，因为它记住了）
````


## 💡 LoRA 和 QLoRA 到底是啥？

### LoRA：给大模型配个“小助手”

- **是什么**：不改变原来的大模型，在旁边加个“小助手”（可训练的参数）
- **好处**：训练参数只有原来的0.1%-1%，**省显存、省时间**
- **效果**：能达到全量微调90%以上的效果

### QLoRA：LoRA的“压缩版”

- **是什么**：先把大模型压缩到4-bit（像把高清电影压成小格式），再加LoRA小助手
- **好处**：**显存需求再降75%**，普通显卡也能跑大模型
- **代价**：效果比LoRA稍微差一点点，但够用

### 两者关系

> **QLoRA = LoRA + 压缩技术**


## 🚀 怎么跑起来？线上 vs 本地

根据你的硬件条件，有3条路可选：


### ✅ 方案一：本地跑（适合显卡好的玩家）

| 硬件要求 | 能跑的模型 | 训练时间（7B模型） |
|---------|-----------|------------------|
| RTX 4090 (24GB) | 13B-33B模型 | 2-4小时 |
| RTX 3060 (12GB) | 7B-13B模型 | 4-8小时 |
| 你的8GB电脑 | 1.5B以下小模型 | 可能10小时+ |

**本地跑的命令示例**（基于你已会的Ollama环境）：

```bash
# 安装Unsloth（简化版微调工具）
pip install unsloth

# 加载4bit模型（就是QLoRA）
from unsloth import FastLanguageModel
model, tokenizer = FastLanguageModel.from_pretrained(
    model_name="unsloth/qwen2-7b",
    load_in_4bit=True,  # 关键：用4bit压缩
)

# 加LoRA适配器
model = FastLanguageModel.get_peft_model(model, r=16)
```

# 准备数据（用你公司员工手册做的问答对）
# ...开始训练
````
优缺点：
✅ 数据不出本地，隐私安全
✅ 一次投入，无限次使用
❌ 硬件门槛高，你的8GB电脑不推荐
````

数据质量决定微调效果

| 数据质量 | 效果 |
|---------|------|
| ✅ 100条高质量、覆盖全面 | 模型学得准，回答专业 |
| ⚠️ 数据有噪音、不一致 | 模型学歪了，回答混乱 |
| ❌ 数据太少（<50条） | 学了等于没学 |

微调的本质
微调不是让模型学会新语言，而是让它精通你的领域。
就像你有个博士生员工，他已经会读书写字（基础能力），微调就是让他熟悉你们公司的具体业务。

微调的价值
| 价值点 | 说明 |
|--------|------|
| 回答更准 | 模型真正记住了你的知识，不依赖检索 |
| 响应更快 | 不用每次现查资料，直接答 |
| 更像专家 | 说话方式、用词都更像你们公司的人 | 

微调的代价
| 代价 | 说明 |
|------|------|
| 硬件门槛 | 需要显卡，越大的模型越贵 |
| 数据准备 | 要整理成高质量的问答对 |
| 知识更新 | 知识变了必须重新训练 |
